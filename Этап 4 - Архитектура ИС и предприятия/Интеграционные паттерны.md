## Ключевые концепции

### Integration styles (Hohpe & Woolf — *Enterprise Integration Patterns*)
- **File Transfer** — batch files
- **Shared Database** — anti-pattern, но common in legacy
- **Remote Procedure Invocation** — sync API calls
- **Messaging** — async, loose coupling

### Enterprise Integration Patterns (selection)
- Message Channel, Message Router, Content-Based Router
- Message Translator, Canonical Data Model
- Message Endpoint, Polling Consumer, Event-Driven Consumer
- Process Manager (Saga orchestrator)

### Integration technologies
| Pattern | Technology |
|---------|------------|
| Point-to-point | REST, gRPC |
| ESB | MuleSoft, WSO2 (legacy trend) |
| iPaaS | Zapier, Workato, Boomi |
| Event streaming | Kafka as integration backbone |
| API Management | Kong, Apigee, AWS API Gateway |

### API-led connectivity (MuleSoft)
- **System APIs** — access core systems
- **Process APIs** — compose system APIs
- **Experience APIs** — tailored for channels

### Challenges
- Legacy system constraints
- Data format transformation
- Semantic mismatch between systems
- SLA differences
- Security (B2B, API keys, mTLS)
