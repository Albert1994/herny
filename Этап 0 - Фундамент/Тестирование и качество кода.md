# Тестирование и качество кода

**Этап:** [[00 Обзор этапа|0 — Фундамент]]  
**Время:** ~1 неделя

---

## Цель

Строить архитектуру, которую можно безопасно менять — тесты и метрики качества это обеспечивают.

---

## Ключевые концепции

### Пирамида тестирования
- **Unit** — изолированная логика, быстрые, много
- **Integration** — взаимодействие компонентов (DB, API)
- **E2E** — сквозные сценарии, медленные, мало
- Contract tests — для микросервисов (Pact)

### Принципы
- Testable design — DI, чистые функции, отсутствие hidden dependencies
- TDD как инструмент проектирования (Red-Green-Refactor)
- Test doubles: mock, stub, fake, spy

### Качество кода
- Code coverage — ориентир, не цель
- Static analysis (SonarQube, ESLint, Roslyn analyzers)
- Cyclomatic complexity
- Code review checklist

## Практика

1. Написать unit-тесты на доменную логику до реализации (TDD)
2. Integration test с testcontainers (PostgreSQL in Docker)
3. Настроить CI: lint + tests на каждый PR

---

## Критерии освоения

- [ ] Различаю unit, integration, e2e и знаю пропорции
- [ ] Могу протестировать код с внешними зависимостями через DI
- [ ] CI падает при failing tests — не merge без green build
