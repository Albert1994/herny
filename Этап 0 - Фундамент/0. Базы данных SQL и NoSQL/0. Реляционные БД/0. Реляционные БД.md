## Главная мысль

Реляционная БД — это строгая схема, ACID-транзакции и JOIN как основной инструмент моделирования связей. Сила SQL — предсказуемость и целостность данных; цена — жёсткость схемы и сложность горизонтального масштабирования.

## Когда выбирать SQL

- Данные связаны (users → orders → payments) и нужна целостность через FK
- Нужны транзакции между несколькими таблицами
- Запросы сложные: агрегации, отчёты, ad-hoc аналитика
- Схема относительно стабильна

**Типичные СУБД:** PostgreSQL, MySQL, SQLite, Oracle, SQL Server.
