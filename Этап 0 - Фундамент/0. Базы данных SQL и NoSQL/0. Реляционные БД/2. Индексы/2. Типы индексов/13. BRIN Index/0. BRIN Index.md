# 13. BRIN Index

**Раздел:** [[../0. Типы индексов|Типы индексов]]  
**Контекст:** [[../../0. Индексы|Индексы]]

---

## Коротко

**BRIN** — Block Range Index.

Он хранит не каждую строку, а summary по диапазонам блоков таблицы: например min/max значений.

---

## Для чего нужно

BRIN нужен для огромных таблиц, где данные физически коррелируют с колонкой.

Типичный пример:

- логи;
- events;
- time-series;
- append-only таблицы;
- `created_at` растёт вместе с физическим порядком вставки.

---

## Как выглядит

```text
blocks 1-128:   created_at min=10:00 max=10:05
blocks 129-256: created_at min=10:05 max=10:10
blocks 257-384: created_at min=10:10 max=10:15
```

Индекс очень маленький.

---

## Пример

```sql
CREATE INDEX idx_logs_created_brin
ON logs USING BRIN (created_at);
```

Запрос:

```sql
SELECT * FROM logs
WHERE created_at BETWEEN '2026-06-17 10:06'
                    AND '2026-06-17 10:08';
```

BRIN отбрасывает блоки, min/max которых точно не пересекаются с диапазоном.

---

## Как работает вставка

Новые строки добавляются в блоки таблицы. BRIN обновляет summary для диапазона блоков.

Это намного дешевле, чем добавлять каждую строку в большой B-tree.

---

## Ограничения

BRIN плохо работает, если данные физически перемешаны.

Если `created_at` случайно распределён по всей таблице, min/max каждого блока будет широким, и BRIN почти ничего не отсеет.

---

## Как объяснить на интервью

BRIN хранит summary по диапазонам физических блоков таблицы. Он очень маленький и хорошо подходит для огромных append-only таблиц, где значение колонки коррелирует с физическим порядком строк.
