# 12. SP-GiST Index

## Коротко

**SP-GiST** — Space-Partitioned GiST.

Это PostgreSQL access method для структур, которые делят пространство на непересекающиеся области.

SP-GiST подходит для:

- tries;
- quadtrees;
- k-d trees;
- IP ranges;
- некоторых spatial задач.

---

## Отличие от GiST

GiST часто группирует объекты через bounding boxes, которые могут пересекаться.

SP-GiST делит пространство на части, которые обычно не пересекаются.

```text
GiST:     overlapping bounding boxes
SP-GiST:  partitioned space
```

---

## Для чего нужно

SP-GiST полезен, когда данные естественно разбиваются:

- пространство на квадранты;
- строки по префиксам;
- IP-адреса по префиксам;
- точки по координатам.

---

## Пример PostgreSQL

```sql
CREATE INDEX idx_points_spgist
ON points USING SPGIST (location);
```

Для некоторых типов запросов SP-GiST может быть быстрее GiST, особенно если partitioning хорошо соответствует данным.

---

## Как работает вставка

1. БД смотрит на значение.
2. Определяет, в какую область/partition оно попадает.
3. Спускается по partition tree.
4. При необходимости создаёт новые узлы разбиения.

---

## Как работает получение

Запрос задаёт область или условие. SP-GiST обходит только те partition, которые могут содержать ответ.

Для IP prefix это похоже на trie: идём по битам адреса.

---

## Как объяснить на интервью

SP-GiST — PostgreSQL access method для partitioned search trees. В отличие от GiST, который часто работает с пересекающимися bounding boxes, SP-GiST делит пространство на непересекающиеся части: quadtree, trie, k-d-like структуры.
