# 8. Operator Class и Collation Index

**Раздел:** [[../0. Типы индексов|Типы индексов]]

---

## Коротко

**Operator class** говорит индексу, какие операторы он поддерживает и как сравнивать значения.

**Collation** определяет правила сортировки и сравнения строк.

Это важно, потому что «строки сравниваются» не всегда означает одно и то же.

---

## Для чего нужно

B-tree по тексту может использоваться для `=` и сортировки, но для некоторых `LIKE`-запросов нужен специальный operator class.

Пример PostgreSQL:

```sql
CREATE INDEX idx_users_name_pattern
ON users (name text_pattern_ops);
```

Для:

```sql
SELECT * FROM users WHERE name LIKE 'Alex%';
```

---

## Где используется

- `LIKE 'prefix%'`;
- case-sensitive/case-insensitive сравнение;
- разные языковые collation;
- `citext`;
- trigram operator classes;
- vector operator classes: cosine, L2, inner product.

---

## Пример с pg_trgm

```sql
CREATE EXTENSION IF NOT EXISTS pg_trgm;

CREATE INDEX idx_users_name_trgm
ON users USING GIN (name gin_trgm_ops);
```

Здесь `gin_trgm_ops` — operator class, который говорит GIN-индексу, как индексировать строку через триграммы.

---

## Как работает получение

Оптимизатор смотрит не только на колонку, но и на оператор:

```sql
WHERE name LIKE 'Alex%'
WHERE name ILIKE '%alex%'
WHERE embedding <=> query_vector
```

Для каждого оператора нужен индекс, который умеет его поддерживать.

---

## Частая ошибка

Создать обычный индекс:

```sql
CREATE INDEX ON users(name);
```

и ожидать, что он поможет для:

```sql
WHERE name ILIKE '%alex%'
```

Обычно не поможет. Нужен trigram index.

---

## Как объяснить на интервью

Operator class определяет, какие операторы и правила сравнения поддерживает индекс. Collation влияет на сортировку и сравнение строк. Поэтому для `LIKE`, `ILIKE`, trigram и vector search часто нужны специальные operator classes, а не просто индекс по колонке.
