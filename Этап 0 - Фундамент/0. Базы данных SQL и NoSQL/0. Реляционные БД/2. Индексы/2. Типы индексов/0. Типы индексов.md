## Главная мысль

Тип индекса выбирается под форму данных и форму запроса.

Нельзя заменить все индексы одним B-tree:

- для JSONB и массивов нужен GIN;
- для геоданных — GiST/SP-GiST/spatial;
- для огромных append-only таблиц — BRIN/zone maps;
- для поиска по подстроке — trigram;
- для embeddings — vector index;
- для full-text — inverted index.

## Как читать этот раздел

1. Сначала изучи классические индексы: primary/unique, foreign key, composite, partial, covering.
2. Потом PostgreSQL access methods: B-tree, Hash, GIN, GiST, SP-GiST, BRIN.
3. Затем специализированные: trigram, full-text, spatial, vector, bitmap, zone maps.
4. После каждого типа задавай вопрос: «какой запрос этот индекс ускоряет и какую цену он добавляет на запись?»
