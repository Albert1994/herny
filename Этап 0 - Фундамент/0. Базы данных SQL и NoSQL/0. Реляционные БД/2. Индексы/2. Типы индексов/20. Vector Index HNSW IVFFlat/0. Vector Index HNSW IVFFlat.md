## Коротко

**Vector index** ускоряет поиск похожих векторов: embeddings.

Это основа semantic search, RAG, поиска похожих товаров, изображений и документов.

Обычный B-tree не подходит, потому что нужно искать не равенство, а ближайших соседей в многомерном пространстве.

---

## Для чего нужно

Запросы:

```text
найди документы, похожие на вопрос пользователя
найди товары, похожие на этот товар
найди изображения, похожие на это изображение
```

В SQL с pgvector:

```sql
SELECT *
FROM documents
ORDER BY embedding <=> '[...]'
LIMIT 10;
```

`<=>` — cosine distance operator.

---

## Основные структуры

| Индекс | Идея | Когда использовать |
|---|---|---|
| HNSW | граф ближайших соседей | высокая recall/скорость, больше памяти |
| IVFFlat | разбиение на кластеры | большие данные, нужен training/analyze |
| IVF/PQ | кластеры + сжатие | очень большие векторные коллекции |
| DiskANN | graph + диск | когда данные не помещаются в RAM |
| Annoy | forest of random projection trees | read-heavy approximate search |
| FAISS | библиотека с IVF/HNSW/PQ/GPU индексами | ML/AI vector search вне SQL |

---

## Где встречается

- PostgreSQL + pgvector: HNSW, IVFFlat.
- FAISS: IVF, PQ, HNSW, GPU-поиск.
- Annoy: approximate search через random projection trees.
- Milvus, Qdrant, Weaviate, Pinecone: managed/vector DB индексы.
- DiskANN-подобные подходы: когда индекс частично живёт на диске.

## pgvector HNSW пример

```sql
CREATE EXTENSION IF NOT EXISTS vector;

CREATE INDEX idx_documents_embedding_hnsw
ON documents
USING hnsw (embedding vector_cosine_ops);
```

Запрос:

```sql
SELECT id, title
FROM documents
ORDER BY embedding <=> $query_embedding
LIMIT 10;
```

---

## pgvector IVFFlat пример

```sql
CREATE INDEX idx_documents_embedding_ivf
ON documents
USING ivfflat (embedding vector_cosine_ops)
WITH (lists = 100);
```

Для IVFFlat важно иметь данные до построения индекса и настроить probes:

```sql
SET ivfflat.probes = 10;
```

---

## Как работает HNSW

HNSW строит многослойный граф.

```text
верхний слой: быстрые дальние переходы
нижний слой: подробный граф соседей
```

Поиск:

1. стартуем с некоторой точки;
2. жадно идём к более близким соседям;
3. спускаемся по слоям;
4. на нижнем слое собираем top-k кандидатов.

---

## Как работает вставка

Для нового вектора HNSW:

1. ищет ближайшие существующие узлы;
2. добавляет новый узел в граф;
3. создаёт связи с соседями;
4. иногда добавляет узел на верхние уровни.

Вставка дороже, чем в простой B-tree, но поиск намного быстрее полного перебора.

---

## Approximate nearest neighbor

Vector indexes часто дают **приближённый** ответ.

Trade-off:

- быстрее поиск;
- меньше чтений;
- но можно пропустить идеального ближайшего соседа.

Метрика качества — recall.

---

## Как объяснить на интервью

Vector index нужен для nearest-neighbor search по embeddings. HNSW строит граф похожих векторов и быстро ищет ближайших соседей, IVFFlat делит пространство на кластеры и ищет только в части кластеров. Это approximate search: мы меняем идеальную точность на скорость.
