# 18. Bitmap Index

**Раздел:** [[../0. Типы индексов|Типы индексов]]  
**Контекст:** [[../../0. Индексы|Индексы]]

---

## Коротко

**Bitmap index** хранит битовую карту для значения.

```text
status = 'new'  -> 1 0 0 1 0
status = 'paid' -> 0 1 0 0 1
```

Каждый бит соответствует строке или позиции.

---

## Для чего нужно

Bitmap index хорошо работает в OLAP для колонок с низкой кардинальностью:

- статус;
- пол;
- категория;
- флаг;
- страна, если значений немного относительно строк.

---

## Как работает получение

Запрос:

```sql
WHERE status = 'paid'
  AND is_active = true
```

Система берёт две bitmap:

```text
status='paid'  -> 0 1 0 0 1
is_active=true -> 1 1 0 1 1
AND            -> 0 1 0 0 1
```

Битовые операции очень быстрые.

---

## Как работает вставка/обновление

При вставке нужно добавить бит в карты соответствующих значений.

При update, например `status = 'new'` → `paid`, нужно изменить несколько bitmap.

В OLTP с частыми update это может быть дорого.

---

## Где используется

- Oracle Bitmap Index;
- columnar DB;
- OLAP engines;
- query execution внутри PostgreSQL может использовать Bitmap Index Scan, но это не то же самое, что постоянный bitmap index.

---

## Bitmap Index Scan в PostgreSQL

PostgreSQL может строить bitmap во время выполнения запроса:

```text
Bitmap Index Scan -> Bitmap Heap Scan
```

Это runtime-стратегия для комбинирования условий, а не отдельный persistent bitmap index как в Oracle.

---

## Как объяснить на интервью

Bitmap index хранит битовую карту для каждого значения и очень быстро комбинирует условия через AND/OR. Он хорош для OLAP и низкой кардинальности, но плохо подходит для частых update в OLTP.
