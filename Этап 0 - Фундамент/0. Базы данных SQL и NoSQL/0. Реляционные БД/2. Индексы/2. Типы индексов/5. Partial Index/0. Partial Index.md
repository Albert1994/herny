# 5. Partial Index

**Раздел:** [[../0. Типы индексов|Типы индексов]]  
**Контекст:** [[../../0. Индексы|Индексы]]

---

## Коротко

**Partial index** — индекс только на часть строк таблицы.

```sql
CREATE INDEX idx_active_users_email
ON users(email)
WHERE status = 'active';
```

Он меньше полного индекса и быстрее обновляется.

---

## Для чего нужно

Partial index нужен, когда запросы почти всегда работают с одним подмножеством данных:

- только активные пользователи;
- только неоплаченные заказы;
- только незакрытые тикеты;
- только `deleted_at IS NULL`;
- только `status = 'pending'`.

---

## Пример

```sql
CREATE INDEX idx_orders_pending_created
ON orders(created_at)
WHERE status = 'pending';
```

Запрос:

```sql
SELECT *
FROM orders
WHERE status = 'pending'
ORDER BY created_at
LIMIT 100;
```

Индекс содержит только pending-заказы, а не всю таблицу.

---

## Как работает вставка

При вставке строки БД проверяет predicate индекса:

```text
status = 'pending' ?
```

- Если да — строка добавляется в индекс.
- Если нет — индекс не трогается.

При `UPDATE` строка может войти в индекс или выйти из него.

---

## Как работает получение

Оптимизатор использует partial index, если может доказать, что условие запроса соответствует predicate индекса.

Индекс:

```sql
WHERE status = 'pending'
```

Запрос должен явно или логически содержать:

```sql
WHERE status = 'pending'
```

---

## Частая ошибка

Создать partial index, но писать запрос иначе:

```sql
WHERE status IN ('pending')
```

или через параметр, из-за которого оптимизатор не всегда может доказать совпадение predicate.

---

## Как объяснить на интервью

Partial index индексирует только строки, удовлетворяющие условию. Это уменьшает размер индекса и цену записи. Он полезен, когда основные запросы работают с небольшим и стабильным подмножеством таблицы.
