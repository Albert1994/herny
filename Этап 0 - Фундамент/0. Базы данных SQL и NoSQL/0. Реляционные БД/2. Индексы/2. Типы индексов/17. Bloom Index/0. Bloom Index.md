# 17. Bloom Index

**Раздел:** [[../0. Типы индексов|Типы индексов]]  
**Контекст:** [[../../0. Индексы|Индексы]]

---

## Коротко

**Bloom index** использует идею Bloom filter: он может сказать «значения точно нет» или «значение возможно есть».

В PostgreSQL есть расширение `bloom`.

---

## Для чего нужно

Bloom index полезен для широких таблиц, когда есть много колонок и запросы фильтруют по разным комбинациям равенств.

Вместо множества B-tree индексов можно создать один bloom index.

---

## Пример PostgreSQL

```sql
CREATE EXTENSION IF NOT EXISTS bloom;

CREATE INDEX idx_users_bloom
ON users USING bloom (country, city, status, role);
```

Запрос:

```sql
SELECT * FROM users
WHERE country = 'RU'
  AND status = 'active';
```

---

## Как работает

Bloom filter хранит битовую сигнатуру значений.

При поиске:

- если биты не совпали — строки точно нет;
- если совпали — строка возможно подходит, нужна перепроверка.

То есть bloom index допускает false positives, но не false negatives.

---

## Как работает вставка

1. Для значений колонок считаются несколько hash-функций.
2. Соответствующие биты выставляются в сигнатуре.
3. Сигнатура сохраняется в индексе.

---

## Ограничения

- Только равенство.
- Возможны false positives.
- Не подходит для range/order.
- Обычно нишевый инструмент, не замена нормальному проектированию индексов.

---

## Как объяснить на интервью

Bloom index использует вероятностную структуру: он быстро отсеивает строки, которые точно не подходят, но может вернуть кандидатов с false positives. Полезен для широких таблиц и запросов по разным комбинациям равенств.
