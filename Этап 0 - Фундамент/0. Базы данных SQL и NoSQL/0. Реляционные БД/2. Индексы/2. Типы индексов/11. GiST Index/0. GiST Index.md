# 11. GiST Index

## Коротко

**GiST** — Generalized Search Tree.

Это PostgreSQL access method для деревьев, где обычный порядок `<`, `=`, `>` недостаточен.

GiST часто используют для:

- геоданных;
- range types;
- nearest-neighbor;
- exclusion constraints.

---

## Для чего нужно

B-tree хорошо сравнивает числа и строки. Но как индексировать пересечение прямоугольников, периодов бронирования или расстояние до точки?

GiST позволяет описать специальные правила:

- может ли ветка содержать ответ;
- как объединять значения в узле;
- какую ветку выбрать при вставке;
- как разделить переполненный узел.

---

## Range пример

```sql
CREATE TABLE bookings (
  id bigserial PRIMARY KEY,
  room_id bigint NOT NULL,
  period tsrange NOT NULL
);

CREATE INDEX idx_bookings_period_gist
ON bookings USING GIST (period);
```

Запрос:

```sql
SELECT * FROM bookings
WHERE period && tsrange('2026-06-01', '2026-06-10');
```

`&&` означает пересечение диапазонов.

---

## Как работает получение

1. Внутренние узлы хранят обобщённое значение, например покрывающий диапазон.
2. Функция `consistent` говорит, может ли ветка содержать ответ.
3. Ненужные ветки отбрасываются.
4. На листьях проверяется точное условие.

---

## Как работает вставка

GiST выбирает ветку с минимальной `penalty`: куда новое значение добавить дешевле всего.

Если узел переполнен, `picksplit` решает, как разделить значения.

---

## Как объяснить на интервью

GiST — это обобщённый фреймворк деревьев поиска в PostgreSQL. Он нужен для типов, где обычный B-tree не подходит: ranges, geometry, nearest neighbor, exclusion constraints.
