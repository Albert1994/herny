# 1. Primary Key и Unique Index

---

## Коротко

**Primary Key index** и **Unique index** не только ускоряют поиск, но и защищают целостность данных.

- `PRIMARY KEY` гарантирует уникальность и `NOT NULL`.
- `UNIQUE` гарантирует, что значения не повторяются.

Обычно БД автоматически создаёт индекс под `PRIMARY KEY` и `UNIQUE` constraint.

---

## Для чего нужно

Эти индексы отвечают на два вопроса:

1. Как быстро найти строку по идентификатору?
2. Как не дать записать дубликат?

Пример:

```sql
CREATE TABLE users (
  id bigint PRIMARY KEY,
  email text UNIQUE NOT NULL
);
```

Здесь БД создаст индекс по `id` и уникальный индекс по `email`.

---

## Где используется

- `PRIMARY KEY` почти в каждой OLTP-таблице.
- `UNIQUE` для email, username, external_id, номера документа.
- Composite unique для бизнес-правил: «один товар в заказе только один раз».

```sql
CREATE UNIQUE INDEX idx_order_items_unique
ON order_items(order_id, product_id);
```

---

## Как работает вставка

При вставке:

```sql
INSERT INTO users(id, email) VALUES (1, 'alice@example.com');
```

БД делает:

1. Проверяет индекс primary key: нет ли `id = 1`.
2. Проверяет unique index: нет ли такого `email`.
3. Если дубликат найден — операция отклоняется.
4. Если всё хорошо — строка и индексы обновляются.

---

## Как работает получение

```sql
SELECT * FROM users WHERE id = 1;
```

БД идёт в индекс primary key и быстро находит строку.

```sql
SELECT * FROM users WHERE email = 'alice@example.com';
```

БД использует unique index по email. Так как индекс уникальный, оптимизатор знает: максимум одна строка.

---

## Важные нюансы

### NULL в unique index

В PostgreSQL обычный `UNIQUE` допускает несколько `NULL`, потому что `NULL` не равен `NULL`.

Если нужно иначе, используй дополнительные ограничения или `NULLS NOT DISTINCT` в новых версиях PostgreSQL.

### Primary key и clustered storage

В MySQL InnoDB primary key — это clustered index: листья дерева содержат сами строки.

В PostgreSQL primary key не делает таблицу физически отсортированной.

---

## Как объяснить на интервью

Primary key и unique index — это индексы, которые одновременно ускоряют поиск и поддерживают constraint. Primary key гарантирует уникальный ненулевой идентификатор строки, unique index — уникальность выбранного значения или набора значений.
