# 21. Range и Exclusion Index

**Раздел:** [[../0. Типы индексов|Типы индексов]]  
**Контекст:** [[../../0. Индексы|Индексы]]

---

## Коротко

**Range index** ускоряет запросы по диапазонам.

**Exclusion constraint** запрещает конфликтующие значения, например пересекающиеся бронирования.

В PostgreSQL это часто делается через range types + GiST.

---

## Для чего нужно

Задачи:

- найти бронирования, пересекающие период;
- запретить два бронирования одной комнаты на одно время;
- искать активные тарифы по дате;
- проверять интервалы действия правил.

---

## Range types

PostgreSQL имеет типы:

- `int4range`;
- `numrange`;
- `tsrange`;
- `tstzrange`;
- `daterange`.

Пример:

```sql
CREATE TABLE bookings (
  id bigserial PRIMARY KEY,
  room_id bigint NOT NULL,
  period tsrange NOT NULL
);
```

---

## Индекс на диапазон

```sql
CREATE INDEX idx_bookings_period
ON bookings USING GIST (period);
```

Запрос пересечения:

```sql
SELECT *
FROM bookings
WHERE period && tsrange('2026-06-01', '2026-06-10');
```

---

## Exclusion constraint

Запретить пересекающиеся бронирования одной комнаты:

```sql
CREATE EXTENSION IF NOT EXISTS btree_gist;

ALTER TABLE bookings
ADD CONSTRAINT no_room_overlap
EXCLUDE USING GIST (
  room_id WITH =,
  period WITH &&
);
```

Смысл: нельзя вставить строку, где `room_id` равен и `period` пересекается.

---

## Как работает вставка

При вставке нового бронирования БД проверяет GiST-индекс:

1. Есть ли строки с тем же `room_id`?
2. Пересекается ли `period`?
3. Если да — вставка отклоняется constraint'ом.

---

## Как работает получение

Для поиска пересечений GiST отсекает ветки, диапазоны которых не могут пересекаться с запросом.

---

## Как объяснить на интервью

Range index ускоряет запросы по интервалам, а exclusion constraint использует индекс для запрета конфликтующих строк. Классический пример — нельзя забронировать одну комнату на пересекающиеся периоды.
