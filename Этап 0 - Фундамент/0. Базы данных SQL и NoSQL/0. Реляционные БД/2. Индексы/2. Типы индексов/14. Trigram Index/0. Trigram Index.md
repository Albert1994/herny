# 14. Trigram Index

**Раздел:** [[../0. Типы индексов|Типы индексов]]  
**Контекст:** [[../../0. Индексы|Индексы]]

---

## Коротко

**Trigram index** разбивает строку на тройки символов — триграммы.

Он нужен для поиска по подстроке, fuzzy search и `ILIKE '%text%'`.

В PostgreSQL используется расширение `pg_trgm`.

---

## Для чего нужно

Обычный B-tree помогает для:

```sql
WHERE name LIKE 'Alex%'
```

Но не помогает для:

```sql
WHERE name ILIKE '%lex%'
```

Потому что начало строки неизвестно.

Trigram index помогает искать по фрагментам внутри строки.

---

## Пример

```sql
CREATE EXTENSION IF NOT EXISTS pg_trgm;

CREATE INDEX idx_users_name_trgm
ON users USING GIN (name gin_trgm_ops);
```

Запрос:

```sql
SELECT * FROM users
WHERE name ILIKE '%alex%';
```

---

## Как выглядит

Слово:

```text
alex
```

можно представить триграммами:

```text
"  a", " al", "ale", "lex", "ex "
```

Индекс хранит:

```text
"ale" -> строки, где есть эта триграмма
"lex" -> строки, где есть эта триграмма
```

---

## Как работает вставка

1. БД разбивает строку на триграммы.
2. Для каждой триграммы обновляет posting list.
3. Поэтому запись дороже, чем в обычный B-tree.

---

## Как работает получение

Для `%alex%` БД:

1. Получает триграммы запроса.
2. Находит строки-кандидаты по posting lists.
3. Перепроверяет условие `ILIKE`, чтобы убрать ложные совпадения.

---

## Где используется

- поиск по имени;
- fuzzy matching;
- autocomplete с ошибками;
- поиск по email/адресам;
- похожесть строк через `%` operator в PostgreSQL.

---

## Как объяснить на интервью

Trigram index разбивает строки на последовательности из трёх символов и строит по ним inverted index. Он полезен для substring и fuzzy search, где B-tree не помогает, например `ILIKE '%text%'`.
