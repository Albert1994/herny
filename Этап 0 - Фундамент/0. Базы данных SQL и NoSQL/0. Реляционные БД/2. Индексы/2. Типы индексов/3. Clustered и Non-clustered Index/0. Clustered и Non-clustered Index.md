# 3. Clustered и Non-clustered Index

## Коротко

**Clustered index** определяет физический порядок хранения строк.

**Non-clustered index** — отдельная структура, которая хранит ключ и ссылку на строку.

Главная разница: clustered index — это фактически сама таблица, организованная по ключу; non-clustered — дополнительный указатель.

---

## Где используется

- MySQL InnoDB: primary key — clustered B+ Tree.
- SQL Server: можно явно выбрать clustered index.
- PostgreSQL: обычные индексы не clustered; есть команда `CLUSTER`, но она не поддерживает порядок автоматически.

---

## Как выглядит

### Clustered

```text
PK B+ Tree
листья = строки таблицы
```

### Non-clustered

```text
secondary index
key -> primary key / row pointer
```

---

## Пример MySQL InnoDB

```sql
CREATE TABLE orders (
  id bigint PRIMARY KEY,
  user_id bigint NOT NULL,
  created_at timestamp NOT NULL
);
```

В InnoDB строки физически лежат в clustered index по `id`.

Если создать secondary index:

```sql
CREATE INDEX idx_orders_user_id ON orders(user_id);
```

то его листья будут хранить `user_id` и primary key `id`, чтобы потом найти строку в clustered index.

---

## Как работает получение

```sql
SELECT * FROM orders WHERE id = 100;
```

Для clustered primary key:

1. Идём в B+ Tree по `id`.
2. В листе уже лежит строка.

```sql
SELECT * FROM orders WHERE user_id = 42;
```

Для secondary index:

1. Идём в `idx_orders_user_id`.
2. Получаем primary keys подходящих строк.
3. По каждому primary key идём в clustered index.

Это называется lookup/bookmark lookup.

---

## Как работает вставка

Если clustered key монотонный (`bigserial`, `identity`), новые строки обычно добавляются в конец дерева.

Если clustered key случайный (`uuid v4`), вставки распределяются по дереву, могут вызывать page split и фрагментацию.

---

## Практический вывод

Для InnoDB primary key должен быть:

- коротким;
- стабильным;
- желательно монотонным;
- не слишком широким, потому что secondary indexes хранят primary key в листьях.

---

## Как объяснить на интервью

Clustered index определяет физическую организацию таблицы: листья индекса содержат строки. Non-clustered index хранит ключ и ссылку на строку. В InnoDB primary key — clustered index, поэтому выбор primary key влияет на все secondary indexes.
