# 16. Spatial и KNN Index

## Коротко

**Spatial index** ускоряет запросы по геометрии и географии.

**KNN index** помогает искать ближайших соседей: nearest neighbor.

В PostgreSQL/PostGIS часто используются GiST и SP-GiST.

---

## Для чего нужно

Примеры запросов:

- найти кафе в радиусе 1 км;
- найти ближайшего курьера;
- найти полигоны, пересекающие область;
- найти дороги рядом с точкой;
- отсортировать по расстоянию.

---

## PostGIS пример

```sql
CREATE INDEX idx_places_location
ON places USING GIST (location);
```

Запрос:

```sql
SELECT *
FROM places
WHERE ST_DWithin(
  location,
  ST_MakePoint(37.6173, 55.7558)::geography,
  1000
);
```

---

## KNN пример

```sql
SELECT *
FROM places
ORDER BY location <-> ST_MakePoint(37.6173, 55.7558)
LIMIT 10;
```

Оператор `<->` означает distance operator, который может использовать индекс для nearest-neighbor.

---

## Как работает получение

Spatial index обычно сначала работает с bounding boxes:

1. Отбрасывает области, которые точно не пересекаются с запросом.
2. Возвращает кандидатов.
3. Точная геометрическая функция перепроверяет результат.

---

## Как работает вставка

1. Вычисляется bounding box объекта.
2. Объект помещается в подходящую ветку spatial tree.
3. Внутренние bounding boxes обновляются.
4. При переполнении узлы делятся.

---

## Geometry vs Geography

- `geometry` — плоская геометрия, хорошо для локальных проекций.
- `geography` — сферическая модель Земли, удобна для широты/долготы и метров.

Выбор влияет на функции, производительность и точность.

---

## Как объяснить на интервью

Spatial index ускоряет геометрические запросы через bounding boxes и пространственные деревья. KNN index позволяет искать ближайшие объекты через distance operator. В PostGIS чаще всего используется GiST/SP-GiST.
