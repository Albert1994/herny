# 15. Full-text и Inverted Index

**Раздел:** [[../0. Типы индексов|Типы индексов]]

---

## Коротко

**Inverted index** хранит не «документ → слова», а «слово → документы».

Это основа full-text search.

```text
"database" -> [doc1, doc5, doc9]
"index"    -> [doc1, doc2]
```

---

## Для чего нужно

Full-text index нужен для поиска по смысловым токенам, а не просто по подстроке.

Он умеет:

- токенизацию;
- нормализацию слов;
- stemming/lemmatization;
- ranking;
- boolean queries;
- phrase/proximity search в поисковых движках.

---

## PostgreSQL пример

```sql
CREATE INDEX idx_docs_fts
ON documents USING GIN (to_tsvector('russian', body));
```

Запрос:

```sql
SELECT *
FROM documents
WHERE to_tsvector('russian', body) @@ to_tsquery('russian', 'индекс & база');
```

---

## Как работает вставка

Документ:

```text
"Индексы ускоряют поиск в базе данных"
```

БД:

1. разбивает текст на токены;
2. приводит слова к нормальной форме;
3. удаляет стоп-слова;
4. добавляет doc_id в posting list каждого термина.

---

## Как работает получение

Запрос:

```text
индекс & база
```

1. Находим posting list для `индекс`.
2. Находим posting list для `база`.
3. Пересекаем списки.
4. Считаем ranking, если нужно.

---

## Где используется

- PostgreSQL FTS;
- Elasticsearch;
- OpenSearch;
- Lucene;
- Solr;
- поиск по документации, статьям, логам.

---

## Trigram vs Full-text

| Вопрос | Trigram | Full-text |
|---|---|---|
| Ищет | фрагменты строк | слова/лексемы |
| Хорошо для | `%text%`, fuzzy | документы, релевантность |
| Понимает язык | нет/слабо | да, через dictionaries |
| Индекс | GIN/GiST trigram | inverted index |

---

## Как объяснить на интервью

Inverted index хранит термин → список документов. Поэтому full-text search быстро находит документы по словам и сочетаниям слов. Это отличается от B-tree: B-tree ищет упорядоченные значения, а inverted index — элементы внутри текста.
