## Коротко

**Zone Map**, **Sparse Index** и **Data Skipping Index** помогают аналитическим системам не читать лишние блоки данных.

Они хранят summary по крупным кускам данных: min/max, null count, bloom filter, set of values.

---

## Для чего нужно

В аналитике таблицы могут занимать терабайты. Главный выигрыш — не читать то, что точно не нужно.

Пример:

```sql
WHERE event_date BETWEEN '2026-06-01' AND '2026-06-07'
```

Если файл/блок содержит даты только за май, его можно пропустить.

---

## Где используется

- ClickHouse data skipping indexes;
- Parquet row group statistics;
- DuckDB zonemaps;
- Snowflake/BigQuery-подобное pruning;
- Cassandra sparse index в SSTable;
- PostgreSQL BRIN как похожая идея.

---

## Как выглядит

```text
row group 1: price min=10 max=20
row group 2: price min=100 max=200
row group 3: price min=300 max=500
```

Запрос:

```sql
WHERE price BETWEEN 110 AND 130
```

Читаем только row group 2.

---

## Как работает вставка

В column-store данные обычно пишутся пачками. Для каждой пачки система считает statistics:

- min/max;
- количество null;
- bloom filter;
- dictionary;
- set для low-cardinality.

---

## Как работает получение

1. Оптимизатор смотрит на predicate запроса.
2. Сравнивает predicate со statistics блоков.
3. Пропускает блоки, которые точно не подходят.
4. Читает только оставшиеся блоки.

---

## Sparse index

Sparse index хранит указатель не на каждую строку, а на диапазоны.

Это дешевле по памяти, но требует, чтобы данные были отсортированы или сгруппированы.

---

## Как объяснить на интервью

Zone map/data skipping index хранит summary по блокам данных и позволяет не читать блоки, которые точно не удовлетворяют запросу. Это особенно важно в column-store и OLAP, где выигрыш даёт не точечный поиск, а сокращение объёма сканирования.
