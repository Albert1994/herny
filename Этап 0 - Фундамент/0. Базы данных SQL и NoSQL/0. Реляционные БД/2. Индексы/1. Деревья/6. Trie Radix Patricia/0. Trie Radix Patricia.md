# Trie, Radix Tree и Patricia

**Раздел:** [[../0. Деревья|Деревья]]  
**Контекст:** [[../../0. Индексы|Индексы]]

---

## Коротко

**Trie** — дерево для ключей, которые можно читать по частям: символам, байтам, битам.

Путь от корня к узлу образует ключ.

Например, слова `car`, `cat`, `dog`:

```text
root
├── c
│   └── a
│       ├── r  (car)
│       └── t  (cat)
└── d
    └── o
        └── g  (dog)
```

**Radix Tree** и **Patricia Trie** — сжатые варианты trie, где длинные цепочки схлопываются.

---

## Для чего нужно

Trie нужен, когда важен не числовой порядок, а общий префикс.

Типичные задачи:

- autocomplete;
- поиск слов по префиксу;
- IP routing;
- словари;
- индексы строковых ключей;
- prefix compression.

---

## Где используется

- IP-маршрутизация: longest prefix match.
- Поисковые подсказки.
- Словари и spell-check.
- Redis и другие in-memory системы в отдельных структурах.
- ART как оптимизированный radix-подход для in-memory DB.

---

## Как выглядит на практике

Добавим слова:

```text
car
cat
cart
```

Trie:

```text
root
└── c
    └── a
        ├── r  (car)
        │   └── t  (cart)
        └── t  (cat)
```

Общий префикс `ca` хранится один раз.

---

## Как работает вставка

Вставляем слово `cart`.

1. Начинаем с корня.
2. Берём первый символ `c`.
3. Если ветки `c` нет — создаём.
4. Переходим к `a`, потом `r`, потом `t`.
5. В последнем узле ставим флаг: «здесь заканчивается слово».

Если в дереве уже есть `car`, то при вставке `cart` переиспользуются узлы `c -> a -> r`, добавляется только `t`.

---

## Как работает получение

### Точный поиск

Ищем `cat`:

1. root → `c`.
2. `c` → `a`.
3. `a` → `t`.
4. Проверяем, что в узле `t` стоит флаг конца слова.

Сложность: `O(k)`, где `k` — длина ключа. Она не зависит напрямую от количества слов.

### Поиск по префиксу

Ищем все слова на `ca`:

1. Спускаемся root → `c` → `a`.
2. От узла `a` обходим всех потомков.
3. Получаем `car`, `cart`, `cat`.

---

## Radix Tree

В обычном trie может быть много узлов с одним ребёнком:

```text
root -> c -> a -> r -> t
```

Radix Tree сжимает цепочку:

```text
root -> "car" -> "t"
```

Это экономит память и уменьшает количество переходов.

---

## Patricia Trie

Patricia — частный случай radix tree, часто используется для битовых ключей.

Особенно полезен для IP:

```text
10.0.0.0/8
10.1.0.0/16
10.1.2.0/24
```

Маршрутизатор ищет самый длинный подходящий префикс: **longest prefix match**.

---

## SQL-аналогия

Обычный B+ Tree может помочь с prefix search, если паттерн начинается с фиксированной строки:

```sql
SELECT * FROM products WHERE sku LIKE 'ELEC-%';
```

Но trie мыслит эту задачу естественнее: путь по символам `E -> L -> E -> C -> -` сразу приводит к поддереву всех товаров с таким префиксом.

---

## Ограничения

- Может потреблять много памяти.
- Для произвольных substring-запросов не подходит напрямую.
- Для дисковой СУБД B+ Tree часто проще и универсальнее.
- Лучше всего trie раскрывается в памяти.

---

## Как объяснить на интервью

Trie хранит ключ как путь по символам или байтам. Это делает поиск по ключу и префиксу `O(k)`, где `k` — длина ключа. Radix Tree сжимает цепочки trie, а Patricia часто применяется для битовых префиксов, например в IP routing.
