## Коротко

**K-d Tree** — дерево для точек в k-мерном пространстве.

Оно по очереди делит пространство по разным координатам.

Для 2D:

- первый уровень делит по `x`;
- второй — по `y`;
- третий снова по `x`;
- и так далее.

---

## Для чего нужно

K-d Tree нужен для:

- nearest neighbor search;
- spatial search;
- range query в многомерном пространстве;
- ML/геометрических задач;
- поиска похожих точек при малой/средней размерности.

---

## Как выглядит

Точки:

```text
(2,3), (5,4), (9,6), (4,7), (8,1), (7,2)
```

Дерево может сначала разделить по `x`, потом по `y`:

```text
              x < 7?
             /      \
          y < 4?    y < 2?
          /   \      /   \
       ...   ...   ...   ...
```

Каждый узел — точка и ось разделения.

---

## Как работает вставка

Вставляем точку `(6,5)`.

1. На корне сравниваем по `x`.
2. На следующем уровне сравниваем по `y`.
3. Потом снова по `x`.
4. Доходим до пустого места и добавляем точку.

---

## Как работает получение

### Range query

Найти все точки в прямоугольнике.

1. Проверяем текущую точку.
2. Смотрим, пересекает ли прямоугольник левую/правую область.
3. Обходим только потенциально нужные ветки.

### Nearest neighbor

1. Идём в ветку, где должна быть точка запроса.
2. Запоминаем лучшего кандидата.
3. Возвращаясь назад, проверяем: может ли другая ветка содержать точку ближе?
4. Если может — обходим её.

---

## Ограничения

K-d Tree плохо масштабируется на очень большую размерность. В embeddings на сотни/тысячи измерений чаще используют HNSW, IVF/PQ и другие ANN-структуры.

---

## Как объяснить на интервью

K-d Tree — бинарное дерево для k-мерных точек. На каждом уровне оно делит пространство по одной координате. Хорошо подходит для spatial/range/nearest-neighbor задач в малой и средней размерности, но страдает от curse of dimensionality.
