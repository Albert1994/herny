## Коротко

**Interval Tree** и **Segment Tree** — структуры для работы с интервалами.

Они отвечают на вопросы:

- какие интервалы пересекают точку;
- какие интервалы пересекают другой интервал;
- как быстро агрегировать значения по диапазону.

---

## Для чего нужно

Типичные задачи:

- бронирования;
- календарь;
- расписания;
- пересечение временных интервалов;
- тарифы с датами действия;
- range aggregation.

---

## Interval Tree

Interval Tree хранит интервалы:

```text
[10, 20]
[15, 25]
[30, 40]
```

Запрос:

```text
какие интервалы пересекают [18, 19]?
```

Ответ:

```text
[10,20], [15,25]
```

---

## Как работает вставка

Вставляем интервал `[15,25]`.

1. Сравниваем начало интервала с ключом узла.
2. Идём влево/вправо как в BST.
3. В узлах поддерживаем дополнительное поле `max_end` — максимальный конец интервала в поддереве.

`max_end` помогает отбрасывать ветки при поиске пересечений.

---

## Как работает получение

Ищем интервалы, пересекающие `[x1,x2]`.

1. Проверяем текущий интервал.
2. Если левое поддерево имеет `max_end < x1`, оно точно не пересекается — пропускаем.
3. Иначе идём влево.
4. Также проверяем правую ветку, если она может содержать пересечение.

---

## Segment Tree

Segment Tree чаще используют для агрегатов по диапазонам:

```text
sum/min/max на отрезке массива
```

Пример:

```text
array: [5, 2, 7, 1, 3]
query: sum[1..3]
```

Segment tree хранит агрегаты по сегментам.

---

## В БД на практике

В PostgreSQL для интервалов чаще используют:

- range types;
- GiST index;
- exclusion constraints.

То есть концептуально это похоже на interval indexing, но реализовано через механизмы PostgreSQL.

---

## Как объяснить на интервью

Interval Tree ускоряет поиск пересекающихся интервалов, храня в поддеревьях максимум правой границы. Segment Tree ускоряет агрегаты и запросы по диапазонам. В SQL-практике похожие задачи часто решаются range types + GiST.
