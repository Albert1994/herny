# R-Tree и R-star Tree

**Раздел:** [[../0. Деревья|Деревья]]  
**Контекст:** [[../../0. Индексы|Индексы]]

---

## Коротко

**R-Tree** — дерево для пространственных данных: точек, прямоугольников, полигонов, линий.

Обычный B+ Tree хорошо работает с одномерным порядком: числа, даты, строки. Но геоданные двумерные или многомерные.

R-Tree индексирует не одно значение, а область — **bounding box**.

---

## Для чего нужно

R-Tree нужен для запросов вроде:

- найти кафе рядом со мной;
- найти заказы в зоне доставки;
- найти дороги, пересекающие область;
- найти здания внутри полигона;
- nearest neighbor search.

Без spatial index база должна проверять геометрию каждой строки.

---

## Где используется

- PostGIS через GiST/SP-GiST индексы.
- Геоинформационные системы.
- Карты и доставка.
- CAD/GIS-системы.
- Игры и симуляции, когда нужны пространственные запросы.

---

## Как выглядит

Допустим, у нас есть точки на карте. R-Tree группирует их в прямоугольники:

```text
               [bbox всего города]
              /         |          \
       [bbox A]      [bbox B]     [bbox C]
        /   \          /   \         |
     cafe  shop     park  atm      hotel
```

Каждый внутренний узел хранит прямоугольник, который покрывает все дочерние объекты.

---

## Как работает получение

Запрос:

```sql
SELECT * FROM places
WHERE ST_DWithin(location, ST_MakePoint(37.6, 55.7)::geography, 1000);
```

То есть: найти места в радиусе 1 км.

Шаги:

1. Строим область запроса — круг или bounding box вокруг точки.
2. Смотрим корневые bounding boxes.
3. Если box не пересекается с областью запроса — отбрасываем всю ветку.
4. Если пересекается — спускаемся ниже.
5. На листьях проверяем реальные объекты точной геометрической функцией.

Индекс обычно даёт кандидатов, а точная геометрия проверяется после.

---

## Как работает вставка

Вставляем новую точку `P`.

1. С корня выбираем дочерний bounding box, который придётся меньше всего расширить, чтобы включить `P`.
2. Спускаемся до листа.
3. Добавляем объект в лист.
4. Если лист переполнен — делим его.
5. После вставки обновляем bounding boxes по пути вверх.

Пример:

```text
Было:
[bbox A: север города]
[bbox B: юг города]

Новая точка находится на севере.
Выбираем bbox A, потому что он расширится меньше.
```

## Что такое R-star Tree

**R-star Tree** — улучшение R-Tree.

Он старается уменьшить:

- перекрытие bounding boxes;
- суммарную площадь boxes;
- плохие split-решения.

Почему это важно?

Если boxes сильно перекрываются, запрос вынужден идти сразу в несколько веток. Это ухудшает производительность.

```text
Плохо: boxes сильно пересекаются
+---------+
|   A     |
|   +---------+
|   | B   |   |
+---|-----+   |
    +---------+
```

R-star Tree пытается упаковать области аккуратнее.

---

## Практический пример PostGIS

```sql
CREATE TABLE places (
  id bigserial PRIMARY KEY,
  name text NOT NULL,
  location geography(Point, 4326) NOT NULL
);

CREATE INDEX idx_places_location
ON places USING GIST (location);
```

Запрос:

```sql
SELECT name
FROM places
WHERE ST_DWithin(
  location,
  ST_MakePoint(37.6173, 55.7558)::geography,
  1000
);
```

Индекс помогает быстро отсеять места, которые точно далеко.

---

## Важные ограничения

- Spatial index часто возвращает кандидатов, а не окончательный ответ.
- Производительность зависит от распределения данных.
- Если bounding boxes сильно перекрываются, дерево становится менее эффективным.
- Для обычных числовых диапазонов чаще лучше B+ Tree.

---

## Как объяснить на интервью

R-Tree — это дерево для многомерных данных. Оно группирует объекты по bounding boxes. При запросе база отбрасывает целые ветки, если их box не пересекается с областью поиска. R-star Tree улучшает R-Tree, стараясь уменьшить перекрытие и площадь boxes.
