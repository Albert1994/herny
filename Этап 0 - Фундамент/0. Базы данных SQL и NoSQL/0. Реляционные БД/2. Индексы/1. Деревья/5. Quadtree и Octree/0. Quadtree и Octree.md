# Quadtree и Octree

**Раздел:** [[../0. Деревья|Деревья]]  
**Контекст:** [[../../0. Индексы|Индексы]]

---

## Коротко

**Quadtree** делит 2D-пространство на 4 части.

**Octree** делает то же самое для 3D-пространства, но делит область на 8 частей.

Это удобно, когда данные живут в пространстве: точки на карте, объекты в игре, частицы в симуляции.

---

## Для чего нужно

Quadtree нужен, чтобы не проверять каждый объект в пространстве.

Пример: на карте 1 000 000 точек. Пользователь смотрит только маленькую область. Без индекса нужно проверить все точки. С quadtree можно быстро перейти к нужным квадрантам.

Используется для:

- поиска объектов в прямоугольной области;
- collision detection;
- кластеризации точек на карте;
- spatial partitioning;
- оптимизации рендера.

---

## Где используется

- Игровые движки.
- Карты и геосервисы.
- Компьютерная графика.
- Симуляции.
- Некоторые spatial indexes и in-memory структуры.

В классических RDBMS чаще встретишь R-Tree/GiST, но quadtree важен как базовая идея spatial indexing.

---

## Как выглядит

Берём квадрат и делим на 4 квадранта:

```text
+---------+---------+
|   NW    |   NE    |
|         |         |
+---------+---------+
|   SW    |   SE    |
|         |         |
+---------+---------+
```

Если в квадранте слишком много объектов, делим его ещё на 4.

```text
+---------+---------+
| NW      | NE      |
|         |         |
+----+----+---------+
|SW-1|SW-2| SE      |
+----+----+         |
|SW-3|SW-4|         |
+----+----+---------+
```

---

## Как работает вставка

Допустим, в одном листе можно хранить максимум 3 точки.

1. Есть корневая область: весь город.
2. Вставляем точку `(x, y)`.
3. Определяем, в какой квадрант она попадает.
4. Если квадрант ещё лист и в нём есть место — кладём точку туда.
5. Если места нет — делим квадрант на 4 части.
6. Перераспределяем старые точки по новым квадрантам.
7. Вставляем новую точку.

Пример:

```text
В SW уже 3 точки. При вставке 4-й точки SW делится на SW-1, SW-2, SW-3, SW-4.
```

---

## Как работает получение

Запрос: «найти все точки внутри прямоугольника».

1. Начинаем с корня.
2. Проверяем, пересекается ли область узла с областью запроса.
3. Если не пересекается — всю ветку пропускаем.
4. Если пересекается — спускаемся в детей.
5. В листьях проверяем конкретные точки.

Это похоже на R-Tree, но деление пространства фиксированное: всегда на 4 части.

## Пример на практике

Представь карту доставок.

```text
Запрос: показать курьеров в текущем viewport пользователя.
```

Viewport — прямоугольник на карте. Quadtree быстро находит только те квадранты, которые пересекаются с viewport.

Псевдокод:

```text
query(node, rectangle):
  if node.area does not intersect rectangle:
    return []

  if node is leaf:
    return points inside rectangle

  result = []
  for child in node.children:
    result += query(child, rectangle)
  return result
```

## Quadtree против R-Tree

| Вопрос | Quadtree | R-Tree |
|---|---|---|
| Как делит пространство | фиксированно на 4 части | группирует по bounding boxes |
| Хорошо для | равномерного деления карты, игр | произвольных объектов и полигонов |
| Узлы | области пространства | bounding boxes объектов |
| В RDBMS | реже | чаще через GiST/PostGIS |

---

## Как объяснить на интервью

Quadtree рекурсивно делит 2D-пространство на 4 квадранта. Вставка помещает объект в нужный квадрант и делит переполненные области. Поиск обходит только квадранты, пересекающиеся с запросом. Octree — аналогичная идея для 3D, только деление идёт на 8 частей.
