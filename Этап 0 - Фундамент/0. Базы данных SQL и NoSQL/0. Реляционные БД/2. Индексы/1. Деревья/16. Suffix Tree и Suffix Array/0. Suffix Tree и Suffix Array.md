# Suffix Tree и Suffix Array

**Раздел:** [[../0. Деревья|Деревья]]  
**Контекст:** [[../../0. Индексы|Индексы]]

---

## Коротко

**Suffix Tree** и **Suffix Array** — структуры для быстрого поиска подстрок.

Они строятся по всем суффиксам строки.

Для строки `banana` суффиксы:

```text
banana
anana
nana
ana
na
a
```

---

## Для чего нужно

Эти структуры нужны для:

- поиска подстроки;
- pattern matching;
- поиска повторов;
- биоинформатики;
- текстовых движков;
- сжатия и анализа строк.

---

## Suffix Tree

Suffix Tree — компактный trie всех суффиксов.

Если нужно найти подстроку `ana`, мы идём по дереву символов `a -> n -> a`. Если путь есть — подстрока встречается.

---

## Suffix Array

Suffix Array хранит отсортированный массив позиций суффиксов.

Для поиска паттерна можно использовать binary search по суффиксам.

Он обычно компактнее suffix tree.

---

## Как работает вставка

Классические suffix tree/array обычно строят для относительно статичного текста.

При изменениях перестраивать их сложно, поэтому в БД для динамических строк чаще используют:

- trigram index;
- full-text inverted index;
- специализированные поисковые движки.

---

## Как работает получение

Ищем подстроку `ana` в `banana`.

### Suffix Tree

1. Идём по символам `a`, `n`, `a`.
2. Если путь существует — подстрока есть.
3. Потомки узла дают позиции вхождений.

### Suffix Array

1. Делаем binary search по отсортированным суффиксам.
2. Находим диапазон суффиксов, начинающихся с `ana`.
3. Их позиции — вхождения подстроки.

---

## Связь с SQL

В обычных SQL-БД suffix tree редко доступен как стандартный индекс.

Для похожих задач чаще используют:

```sql
WHERE text ILIKE '%pattern%'
```

с trigram index.

Для полнотекста — inverted index.

---

## Как объяснить на интервью

Suffix Tree — компактный trie всех суффиксов строки, а Suffix Array — отсортированный массив суффиксов. Они ускоряют поиск подстрок, но для динамических БД часто заменяются trigram или inverted indexes.
