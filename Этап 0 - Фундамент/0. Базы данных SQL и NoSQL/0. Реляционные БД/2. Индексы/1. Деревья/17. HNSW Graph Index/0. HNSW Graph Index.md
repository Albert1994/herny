## Коротко

**HNSW** — Hierarchical Navigable Small World graph.

Это не дерево, а графовый индекс для approximate nearest neighbor search. Я храню его в разделе структур под индексами, потому что он важен для vector indexes.

---

## Для чего нужно

HNSW нужен для быстрого поиска ближайших векторов:

- semantic search;
- RAG;
- похожие товары;
- похожие изображения;
- рекомендации.

---

## Как выглядит

HNSW строит несколько слоёв графа:

```text
Layer 2:       A -------- Z
Layer 1:    A ---- D ---- Z
Layer 0: A--B--C--D--E--F--...--Z
```

Верхние слои дают быстрые дальние переходы, нижний слой содержит более подробные связи.

---

## Как работает получение

Ищем ближайшие векторы к query vector.

1. Начинаем с entry point на верхнем слое.
2. Переходим к соседу, который ближе к query.
3. Когда улучшений нет — спускаемся на слой ниже.
4. На нижнем слое исследуем кандидатов шире.
5. Возвращаем top-k ближайших.

---

## Как работает вставка

1. Для нового вектора ищутся ближайшие существующие узлы.
2. Новый узел соединяется с несколькими соседями.
3. С некоторой вероятностью он попадает на верхние уровни.
4. Граф локально перестраивается.

---

## Главные параметры

- `M` — сколько связей хранить у узла.
- `ef_construction` — качество построения индекса.
- `ef_search` — качество/скорость поиска.

Чем выше параметры, тем лучше recall, но больше память и время.

---

## Trade-off

HNSW обычно очень быстрый и даёт высокий recall, но:

- потребляет много памяти;
- строится дольше простых индексов;
- даёт приближённый результат;
- требует настройки параметров.

---

## Как объяснить на интервью

HNSW — графовый индекс для approximate nearest neighbor search. Он строит иерархию small-world графов: верхние уровни позволяют быстро приблизиться к нужной области, нижний уровень уточняет ближайших соседей. Это один из основных индексов для vector search.
