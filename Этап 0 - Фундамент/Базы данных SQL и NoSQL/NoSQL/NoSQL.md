# NoSQL

**Раздел:** [[../Базы данных SQL и NoSQL|Базы данных: SQL и NoSQL]]

---

## Главная мысль

NoSQL — не «замена SQL», а набор специализированных моделей данных под конкретные паттерны доступа. Выбираешь не «MongoDB vs PostgreSQL», а «какой паттерн чтения/записи доминирует в задаче».

---

## Семейства NoSQL

| Тип | Примеры | Когда использовать | Конспект |
|-----|---------|-------------------|----------|
| Document | MongoDB, CouchDB | Гибкая схема, nested documents | [[Document DB]] |
| Key-Value | Redis, DynamoDB | Кэш, сессии, counters | [[Key-Value DB]] |
| Column-family | Cassandra, HBase | Write-heavy, time-series | [[Column-family DB]] |
| Graph | Neo4j, Neptune | Связи, рекомендации, fraud | [[Graph DB]] |

---

## SQL vs NoSQL — быстрый выбор

| Критерий | SQL | NoSQL |
|----------|-----|-------|
| Схема | Строгая, заранее | Гибкая или отсутствует |
| Связи | JOIN, FK | Денормализация, вложенность |
| Транзакции | ACID из коробки | Зависит от СУБД |
| Масштабирование | Вертикальное + репликация | Горизонтальное шардирование |
| Запросы | Ad-hoc SQL | Под конкретный access pattern |
