# Document DB

**Раздел:** [[NoSQL]]

---

## Главная мысль

Документ = агрегат. Храни вместе то, что читается вместе.

---

## Модель

Данные хранятся как JSON/BSON-документы с вложенными объектами и массивами.

**Примеры:** MongoDB, CouchDB, Firestore.

---

## Когда использовать

- Схема меняется часто или различается между записями (CMS, каталоги с разными атрибутами)
- Данные естественно вкладываются: `user → addresses[] → orders[]`
- Read по `_id` или по индексированным полям внутри документа

---

## Плюсы и минусы

| Плюсы | Минусы |
|-------|--------|
| Гибкая схема, быстрая разработка | Нет JOIN как в SQL (денормализация или `$lookup`) |
| Горизонтальное шардирование (MongoDB) | Риск дублирования данных |
| Естественная модель для API (JSON) | Сложнее строгая целостность между документами |

---

## Пример структуры

```json
{
  "_id": "user_123",
  "name": "Alice",
  "addresses": [
    { "city": "Moscow", "default": true }
  ],
  "orders": [
    { "id": "ord_1", "total": 1500, "items": [...] }
  ]
}
```
