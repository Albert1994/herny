# Миграции схемы

**Раздел:** [[Реляционные БД|Реляционные БД (SQL)]]

---

## Главная мысль

Схема — это код. Миграции должны быть в git, проходить через CI и применяться в том же порядке на dev, staging и prod.

---

## Что такое миграция

Версионированное изменение структуры БД: добавить колонку, индекс, таблицу, изменить тип.

---

## Принципы безопасных миграций в production

1. **Обратимость** — каждая миграция имеет `up` и `down` (или rollback-план)
2. **Идемпотентность** — повторный запуск не ломает (`IF NOT EXISTS`)
3. **Без downtime** — тяжёлые операции поэтапно:
   - Добавить колонку nullable → заполнить батчами → добавить NOT NULL
   - Создать индекс `CONCURRENTLY` (PostgreSQL), не блокируя запись
   - Переименование: добавить новую колонку → dual-write → переключить чтение → удалить старую
4. **Версионирование** — Flyway, Liquibase, Alembic, Rails migrations — единый порядок на всех средах
