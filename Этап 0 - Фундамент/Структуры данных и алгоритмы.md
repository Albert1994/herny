# Структуры данных и алгоритмы

**Этап:** [[00 Обзор этапа|0 — Фундамент]]  
**Время:** ~2 недели

---

## Цель

Понимать сложность операций и trade-offs структур данных — это напрямую влияет на архитектурные решения (индексы, кэши, выбор хранилища).

---

## Ключевые концепции

### Структуры данных
- Массивы, связные списки, хеш-таблицы, деревья (B-tree, BST), графы
- Stack, Queue, Heap, Priority Queue
- Bloom filter, Skip list — когда применяются в production

### Алгоритмы и сложность
- Big-O: время и память
- Сортировка, поиск, обход графов (BFS, DFS)
- Жадные алгоритмы, divide & conquer

### Связь с архитектурой
| Структура | Где встречается в системах |
|-----------|---------------------------|
| Hash table | Redis, in-memory cache, hash index |
| B-tree | PostgreSQL, MySQL индексы |
| Graph | Social graph, dependency resolution, Neo4j |
| Queue | Message brokers, job queues |
| Heap | Priority queues, top-K queries |
