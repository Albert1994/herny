# Сети и протоколы

**Этап:** [[00 Обзор этапа|0 — Фундамент]]  
**Время:** ~1 неделя

---

## Цель

Понимать, как компоненты системы общаются по сети — основа для проектирования API, микросервисов и интеграций.

---

## Ключевые концепции

### Модель OSI / TCP/IP (практический минимум)
- IP, TCP vs UDP
- DNS, TLS/SSL
- HTTP/1.1, HTTP/2, HTTP/3
- WebSocket — когда нужен persistent connection

### Сетевые паттерны
- Request/Response (sync)
- Long polling, SSE, WebSocket (real-time)
- Connection pooling
- Timeouts, retries, keep-alive

### Безопасность на транспортном уровне
- HTTPS, certificate pinning
- mTLS для service-to-service
- CORS — что это и зачем
