# Принципы проектирования

## Цель

Расширить SOLID практическими принципами, которые применяются ежедневно при проектировании модулей и API.

---

## Ключевые концепции

### Базовые принципы
| Принцип | Суть |
|---------|------|
| **DRY** | Don't Repeat Yourself — одна точка изменения |
| **KISS** | Простое решение лучше «умного» |
| **YAGNI** | Не строй то, что не нужно сейчас |
| **LoD** | Law of Demeter — минимум знаний о внутренностях других модулей |
| **Composition over Inheritance** | Композиция гибче наследования |

### Связность и связанность
- **Cohesion** (связность) — элементы модуля работают на одну цель → высокая
- **Coupling** (связанность) — зависимость между модулями → низкая

### Дополнительные принципы
- **Separation of Concerns** — разделение ответственностей
- **Convention over Configuration**
- **Fail Fast** — раннее обнаружение ошибок
- **Principle of Least Astonishment** — API ведёт себя предсказуемо

## Практика

1. Code review своего кода: найти 3 нарушения DRY и исправить
2. Спроектировать feature с YAGNI — явно записать, что **не** делаем
3. Измерить coupling между модулями (imports graph)
