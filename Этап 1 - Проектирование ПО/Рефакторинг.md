# Рефакторинг

**Этап:** [[00 Обзор этапа|1 — Проектирование ПО]]  
**Время:** ~1 неделя

---

## Цель

Безопасно улучшать структуру кода без изменения поведения — ключевой навык архитектора при эволюции систем.

---

## Ключевые концепции

### Code Smells (Fowler)
- Long Method, Large Class, Feature Envy
- Data Clumps, Primitive Obsession
- Switch Statements, Parallel Inheritance
- Speculative Generality

### Каталог рефакторингов
- Extract Method / Extract Class
- Move Method / Move Field
- Replace Conditional with Polymorphism
- Introduce Parameter Object
- Replace Inheritance with Delegation
- Strangler Fig (на уровне системы — см. [[../Этап 5 - Практика архитектора/Стратегии миграции|Стратегии миграции]])

### Безопасный рефакторинг
1. Characterization tests (если нет тестов)
2. Маленькие шаги
3. Green tests после каждого шага
4. Не смешивать refactor и feature в одном PR

### Technical Debt
- Deliberate vs inadvertent
- Martin Fowler's Technical Debt Quadrant
- Когда платить, когда жить с долгом

## Практика

1. 10 рефакторингов из Fowler на своём коде
2. «Boy Scout Rule» — оставить код лучше, чем нашёл
3. Оценить tech debt в story points для backlog

## Критерии освоения

- [ ] Делаю refactor без изменения behavior (tests green)
- [ ] Называю 10+ code smells и matching refactorings
- [ ] Проект этапа 1 — успешный refactor legacy с тестами
