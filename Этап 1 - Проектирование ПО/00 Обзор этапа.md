## Зачем этот этап

Архитектура начинается не с диаграмм, а с качества проектирования на уровне модулей, классов и API. Плохой design внутри «хорошего» микросервиса — всё равно плохая система.

## Проект этапа: Рефакторинг legacy-модуля

### Задача

Взять open-source проект или свой старый код (500+ строк) и:

- Выявить code smells и антипаттерны
- Применить 3+ паттерна проектирования
- Разбить на модули с чёткими границами
- Покрыть рефакторинг тестами (characterization tests → refactor)

### Критерии сдачи

- [ ] Документ «до/после» с обоснованием каждого изменения
- [ ] Метрики: снижение cyclomatic complexity, рост test coverage
- [ ] PR-style описание для каждого логического коммита
