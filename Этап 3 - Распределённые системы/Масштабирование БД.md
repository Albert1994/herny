---

## Ключевые концепции

### Vertical vs Horizontal scaling
- Scale up (bigger machine) — limit
- Scale out (more machines) — complex but unlimited

### Replication
- **Leader-follower** (primary-replica)
- **Multi-leader** — conflict resolution
- **Leaderless** (Dynamo-style) — quorum reads/writes
- Sync vs async replication — durability vs latency

### Sharding (Partitioning)
- **Range-based** — by ID range
- **Hash-based** — even distribution
- **Directory-based** — lookup table
- Hot spots, resharding, cross-shard queries

### Read scaling
- Read replicas
- Connection pooling (PgBouncer)
- Read/write splitting in application

### Specialized stores
- Time-series (InfluxDB, TimescaleDB)
- Search (Elasticsearch)
- Analytics (ClickHouse, BigQuery)
