## Ключевые концепции

### Messaging patterns
- **Point-to-point** (queue) — one consumer
- **Publish-subscribe** (topic) — many consumers
- **Competing consumers** — scale processing
- **Dead Letter Queue (DLQ)** — failed messages
- **Priority queues**

### Kafka vs RabbitMQ vs SQS
| | Kafka | RabbitMQ | SQS |
|---|-------|----------|-----|
| Model | Log / stream | Broker | Managed queue |
| Ordering | Per partition | Per queue | Best effort |
| Retention | Configurable | Until consumed | 14 days |
| Use case | Event streaming | Task queues | AWS simple async |

### Reliability
- Acknowledgments (manual ack)
- Retry with exponential backoff
- Idempotent processing
- Outbox pattern для exactly-once publish

### Message design
- Schema (Avro, Protobuf, JSON Schema)
- Schema registry
- Message size limits
- Poison messages handling
