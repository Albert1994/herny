## Ключевые концепции

### Cache levels
- **Client-side** — browser, mobile app
- **CDN** — static assets, edge caching
- **Application** — in-process (Caffeine, MemoryCache)
- **Distributed** — Redis, Memcached

### Patterns
- **Cache-aside** (lazy loading) — app manages cache
- **Read-through** — cache loads on miss
- **Write-through** — write to cache + DB together
- **Write-behind** — async write to DB
- **Refresh-ahead** — proactive refresh before expiry

### Invalidation
> «There are only two hard things in CS: cache invalidation and naming things.»
- TTL-based
- Event-based (cache bust on update)
- Versioned keys

### Problems
- **Cache stampede** — thundering herd on expiry
- **Hot keys** — uneven load
- **Stale data** — consistency vs performance
