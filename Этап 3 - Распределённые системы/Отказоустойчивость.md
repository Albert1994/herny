# Отказоустойчивость

---

## Цель

Проектировать системы, которые gracefully degradate при сбоях — patterns resilience из production experience.

---

## Ключевые концепции

### Failure modes
- Hardware, network, software bugs
- Cascading failures
- Partial failures (silent data corruption rare but catastrophic)

### Resilience patterns (Michael Nygard — *Release It!*)
- **Circuit Breaker** — stop calling failing service
- **Bulkhead** — isolate thread pools / connections
- **Timeout** — don't wait forever
- **Retry** — with exponential backoff + jitter
- **Fallback** — degraded response
- **Rate Limiter** — protect downstream

### Distributed transactions
- 2PC — rarely used (blocking)
- **Saga** — compensating transactions
- **TCC** (Try-Confirm-Cancel)

### Design for failure
- Health checks (liveness vs readiness)
- Graceful shutdown
- Chaos engineering (Chaos Monkey, Litmus)
- Multi-AZ / Multi-region

### SLI / SLO / SLA
- **SLI** — metric (availability, latency)
- **SLO** — target (99.9% uptime)
- **SLA** — contract with penalties
- Error budget
