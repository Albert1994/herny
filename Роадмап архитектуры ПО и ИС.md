# Роадмап: архитектура ПО и информационных систем

> Цель: вырасти от уверенного разработчика до архитектора, который проектирует системы с учётом бизнес-контекста, NFR и организационных ограничений.
>
> Ориентир по времени: **12–18 месяцев** при 5–8 часах в неделю.

---

## Как пользоваться

1. Проходи этапы **последовательно** — каждый следующий опирается на предыдущий.
2. В каждой заметке есть блок **«Критерии освоения»** — не переходи дальше, пока не выполнишь их.
3. В конце каждого этапа — **практический проект** (описан в заметке этапа).
4. Отмечай прогресс: `- [ ]` → `- [x]`.

## Метрики прогресса

- **Теория:** % отмеченных тем в чеклистах выше
- **Практика:** 6 завершённых проектов (по одному на этап)
- **Артефакты:** ADR, C4-диаграммы, case studies в портфолио
- **Внешняя валидация:** 3+ mock system design / architecture interviews
