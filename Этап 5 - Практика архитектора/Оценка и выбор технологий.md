## Ключевые концепции

### Technology Radar (Thoughtworks)
- **Adopt** — use in production
- **Trial** — try on real project
- **Assess** — explore, POC
- **Hold** — don't start new, migrate away

### Evaluation criteria
| Критерий | Вопросы |
|----------|---------|
| Fit | Решает ли нашу проблему? |
| Maturity | Production-ready? Community? |
| Team | Есть ли expertise? Learning curve? |
| Ops | Monitoring, deployment, scaling? |
| Cost | License, infra, TCO |
| Lock-in | Migration path if we leave? |
| Security | Compliance, vulnerabilities history |
| Ecosystem | Integrations, tooling |

### POC vs Spike
- **Spike** — time-boxed research, throwaway code
- **POC** — demonstrate feasibility for stakeholders
- Document findings → input for ADR

### Build vs Buy vs Open Source
- Core business → build
- Commodity → buy/SaaS
- Non-differentiating → open source + support

### Fitness Functions (Evolutionary Architecture)
Automated checks that architecture meets constraints:
- «No cyclic dependencies between modules»
- «API response time p99 < 500ms»
- «Test coverage > 80% on domain layer»
